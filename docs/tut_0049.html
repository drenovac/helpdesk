Overview

The following document outlines the application of a software patch to Powerforce. A software update may be applied in two modes

  • Software patch (this document)
  • Running a patch.exe update - run the patch.exe file against your production system (having backed up prior).

Explanation


Powerforce is developed atop a non-SQL application environment called Openinsight by Revelation Technology (USA). The environment is an interpreted programming language, which means that the code does not exist in the form of .dll or .ocx files. The code for the environment and Powerforce is kept in a suite of files represented as *.ov and *.lk files under Windows.

In order to provide a backup environment for the purposes of restoring to a prior version of the "application software", the *.ov & *.lk files within your "OINSIGHT" must be backed up. There is no need to backup more than the *.ov & *.lk files.

Having backed up your current environment, apply the necessary software patch.

 

In the example documented here, the Powerforce application has 2 parts, the Production environment maintained inside the "PForce32" directory and its test environment in "PFtest".

Software updates by Powerforce Software are copied into the sub-directory "Patches", where each patch is held in a further sub-directory.

Backing up Current Application


The quickest and most simplest backup is to:

  • create a sub-directory under your OINSIGHT directory. In the image above, we have created the sub-directory mpd_130415 (15th April 2013, the date of the backup)
  • copy the original oinsight\*.lk and oinsight\*.ov files into the sub-directory.

 

If using DOS, you would execute the following:

  • mkdir mpd_130415
  • cd mpd_130415
  • copy ..\*.ov
  • copy ..\*.lk
  • del rev30010.?? - the two files rev30010.lk and rev30010.ov are a 'temp' file and generally very large, therefore unnecessary for the backup

 

Having created your backup, startup Powerforce and navigate to the option as shown below.

Applying the patch

 

To apply the software patch, you have to set the "Upgrade Location" to the directory directed by Powerforce.

The full path below is to: D:\CORPORATE APPLICATIONS\POWERFORCE\PATCHES\130416


 

Reverting to a prior version of Powerforce


To revert to a prior version of Powerforce, all that is required is to copy "back" the *.ov & *.lk files from the sub-directory into which you have made the backup (as in the above example).

To execute the copy, you must ensure that all users are logged out of Powerforce, as the copy will report file in use errors, and the process cannot be guaranteed to have worked.
