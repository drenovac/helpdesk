Find the Cause


We can we from the export that the export relates to a pay-run.

Open the payrun "1563", through the payroll workbench and open the reports menus, as shown below:

The resultant output will show the GL with the issue, as you can clearly see below, relating to employee id: "VG0824".

Locate the cause

Using the GL option / Code Interpreter, review the setup for the GL cross-reference between the Powerforce code and your external accounting system.

You can confirm that the combination has no external entry, and hence the failure.

Now, source the cause of the GL number used. Check the employee pay class and then verify the classification code details for that pay class, which show:

 

If the GL-entry in the classification code is correct, go back to the GL and create a cross-reference to your external system, otherwise correct the GL entry in the above to point to a GL which has a cross-reference for the external system.
