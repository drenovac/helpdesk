Signon sheet upload is taking too long

Customer has called in that the upload of a 25 line signon sheet is taking too long

See Also

Technote index


Introduction

For all intents and porposes the upload of 12 lines of data is taking an excessive amount of time. The process appears to be working as the status bar is progressing across the screen and the client knows the contents (below).

This is indeed very frustrating that the system appears hung and non-responsive. So, helpdesk request a copy of the .csv file being uploaded.

Upon investigation we find that the upload file is 16MB in size, which is more than you would expect for a 13 line .csv file (it's only text).
So we load it into excel, and indeed confirm that there are 13 lines of text to upload.

However, we then execute a "Ctrl End" button to take us to the end of the .csv file and find the file to be 1,048,576 lines deep.
Clearly this is why the code is still running, it's processing each line to see if there's data there - and hence the excessive upload time.

 

Additional Information

As a result of this issue, helpdesk is recoding the logic for upload to complete as soon as it locates two blank lines one after the other.

You can find further information on this process at "signon sheet processing"
