New function - creating HO admin system

This document details setting up a 2nd copy of Powerforce to manage Staff related payroll

See Also

Technote index

Overview

The client has requested an ability to run Admin payroll through Powerforce.

The general mode that we would implement this in is to create an 'ADMIN' company and allow users to select the company on login.
In this instance the customer wanted very, very secure access, therefore we took a different approach as outlined in this document.

To be certain of absolute access control, it's optimal to return the access control back to the Operating System.


To that effect, we create a copy of the current production environment for Powerforce.
User access control into Powerforce can be managed through Powerforce, however company IT can ensure that only the correct individuals have access to the application in the first place.

Administrative Setup

The nature of the setup is that the production data resides within the "HO" directory structure and as such the Operating system administrators can control file access restrictions to only authorised system logins.
