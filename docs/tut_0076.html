Cannot allocate leave to employee

Employee Number 0130 Moe Quince - he is a part time employee with us and has always been part time but we can now no longer allocate leave to him and when we try it says he is casual. As you can see he is set up as part time.

Introduction

  • To start the investigation confirm the employment type and pay class of the employee in question.
    Then confirm that the pay class (classification code) accrues leave - (as per the image below)
    This quick check confirms that the employee's pay class does have a means to calculate out leave.


  • The next step is to confirm what is being said is the actual issue - the inability to schedule leave.
    The quickest path to this is to open an existing shift for the employee

  • Double click on the shift to 'Edit the booking'
    Then, change the scheduling code to 'Annual Leave" to see what the system is doing

 

  • The message is clear on what the issue is, however, due to the closeness of the pay class ids, the error is not agains the employees' base pay class, rather it's against CA-L2 CAS.
    That can only be as a result of this site, having a classification change, so let's check that

Open the 'client operations' for this site and indeed verify that all Grade 1 Casual (FTE) are to be paid at "CA-L2 CAS".

Finally, let's open the CA-L2 CAS Entitlements details

To which we see that pay class CA-L2 CAS has not leave entitlements - and hence the reason this employee cannot be allocated leave (to this site).

Additional Information

You now have two approaches to correct this

  1. Enter leave entitlement details for the pay class - which will allow you to have specific leave accruals for leave attributable to / at this site.
  2. Create a rostering site called "Leave" and roster the employee to that - to which the pay class will always default at the employee's base pay class.

 

The choice of which option works best is based on the broader definition of the actual award that the employee is covered by.
