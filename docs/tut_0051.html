Problem

Through whatever means a number of rosters have not correctly flagged with their correct invoice number. The problem is best demonstrated through the scheduling workbench's display: Had the data been correct, the system should have reported a valid invoice number.

Therefore we have to reset the necessary flags to enable the shifts to be re-invoiced.

 

 

Batch Change Utility

The following image represents the in-house tool referred to in our introduction.

The few tricks are to NOT enter quotation marks around any value that you place in the 3rd box (where the '01/06/13' appears).
So, in this instance enter the data as you see it above:

The end result that you are looking to execute is:

SELECT ROSTER_TIMECARD WITH ROSTER_DATE BETWEEN '01/06/13' AND '30/06/13'
SELECT ROSTER_TIMECARD WITH CUSTOMER = 'SOCRELIEF'

The goal is to create as few records for the extract as possible, and therefore we start with a date range select of the ROSTER_DATE (it is invoiced, and hence the data result returns just the small sub-set), and then you create the next SELECT which works on the previous result.

 

The above image shows the completed form as is required to complete the data manipulation.

The final entry relating to "Change To" is possibly a piece of information that you shall have to confirm with Powerforce support to confirm for you. In this instance the goal is to force the data element "INVOICE_XREF" to .null. (cleared out, or "").
