Overview

We have found that “Holiday Leave Loading” has been paying out automatically when an employee is on leave.  Please remove this automatic element in Power Force.

We would like this process to stop being automatic and for this to be a manual process as a majority of our employees are not entitled to the 17.5% holiday leave loading. 


The parameters which control an employee's leave accruals and processing are found on the classification (pay class) code that the employee is on at the time of taking leave. As an example:

.

 

Here you have a classificaiton, which has Annual Leave accuring according to the Leave Code "AL-PPT".

When you then look at the details of the leave code itself, it has two entries which contribute to the classification (pay class) giving an employee the loading.

  1. The Holiday Loading Pay Code indicates that this Leave Code pays as per the "201" Allowance code.
  2. The Leave Loading value is set at 17.50 - which overrides the value that may be held on the 201 Allowance code.

 

The Allowance code "201" itself has a default setting of 17.50 as the rate of leave loading in case there was no entry on the Leave Code "AL-PPT".

 

To remove the loading you must remove the "201" entry from the Holiday Loading Pay Code.

If you have employees that are entitled to receive a leave loading, then create a seperate Annual Leave code, and a seperate classification code that links to that particular leave code.

A higher level control on the payment of leave is set at the employment type configuration. Leave accurals has to be confirmed as an entitlement on the employment type for the employee to accrue any hours.
