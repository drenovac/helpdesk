All of a sudden shift recosts are showing up SAT loading on a weekday shift. What is the cause of this?

 

 

Introduction

The shift is a Monday & all of a sudden, the shifts are having a Saturday loading applied - which is incorrect.

 

Step 1

After a few trials and reading the source code, it was found to be a change to the configuration of the allowance and deduction code details for the "SAT" Saturday. You can see that the configuration is indicating to apply this allowance to All hour types - which is incorrect in the use of this configuration. The field has to be changed to "Normal" hours, which will return the system back to normal.
